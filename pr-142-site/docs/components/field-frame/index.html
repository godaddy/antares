FieldFrame

Field frame component with integrated Label, description Text, and FieldError for form fields

Field frame component using React Aria Components Provider pattern with integrated Label, description Text, and FieldError. Renders the shared frame (label, border, description, error) around field content and manages accessibility.

Features

  • Visual frame: Shared border, focus, disabled, and invalid styling around field content
  • Focus delegation: Tracks focus from any focusable child (input, button) via data-focus-within and data-focus-visible
  • Label, description, error: Optional Label, helper Text, and FieldError with proper accessibility association
  • Required indicator: Shows asterisk next to label when the field is required
  • React Aria integration: Uses Provider pattern so nested elements stay in sync with the frame
  • Field wrapper composition: Designed to sit inside TextField, NumberField, DateField, or Select as the main field UI

Component composition

Internal structure

FieldFrame renders a single React Aria Provider that supplies the necessary context to its children. Structure tree:

The following list describes the internal React Aria components used to compose FieldFrame:

  • Label (optional) — Rendered when label is provided. Shows a required indicator (asterisk).
  • Group — The visual frame. Exposes data-focus-within, data-focus-visible, data-disabled, and data-invalid. All focusable children (input, button, etc.) live inside this Group.
  • FieldError (optional) — Rendered when errorMessage is provided. Rendered below the Group, above the description. Associated with the field for accessibility.
  • Text (optional) — Description block rendered when description is provided.

Children you pass to FieldFrame (e.g. Input, Button, DateInput, SelectValue) are rendered inside the Group. The provider ensures nested Input and Button read the same disabled/required state as the frame.

States

The states disabled, readonly, and invalid are applied when the field wrapper (e.g. TextField, NumberField, Select) is marked as disabled, readonly, or invalid respectively. The frame reflects these and other states via data attributes on the group.

StateCSS Selector
disabled — Applied when the field wrapper is disabled.[data-disabled]
invalid — Applied when the field wrapper is invalid (e.g. validation failed).[data-invalid]
readonly — Applied when the field wrapper is read only.[data-readonly]
focus — Whether an element within the frame has keyboard focus (for focus-ring styling).[data-focus-visible]
focus-within — Whether an element within the frame is focused, via mouse or keyboard.[data-focus-within]
