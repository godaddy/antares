Drawer

An overlay panel that slides in from a screen edge, supporting sidebar navigation and bottom sheets.

An overlay panel that slides in from a screen edge. Built like Modal (a RAC modal overlay with an elevation="overlay" panel and CSS transitions), it supports sidebar navigation and bottom sheets.

DrawerTrigger is the composition helper: it wraps a trigger button and a <Drawer>, handling open state and aria-controls linkage automatically. Based on RAC DialogTrigger.

Features

  • Slides in from any screen edge (top, bottom, left, right)
  • overlay elevation panel over a dimmed backdrop, matching Modal
  • Physical placement (does not flip for RTL)
  • Focus trapping, backdrop dismiss, and scroll lock via RAC Modal
  • CSS transitions with prefers-reduced-motion support

Examples

Basic Usage

Use <DrawerTrigger> to wrap a trigger button and a <Drawer>. The drawer opens when the trigger is pressed.

Bottom Sheet

Use placement="bottom" with controlled state for a bottom sheet pattern.

Placements

The drawer slides in from whichever edge you set with placement.

Nested Popover

Popovers rendered inside a drawer stay open without dismissing the drawer.

No Escape Dismiss

Use isKeyboardDismissDisabled to prevent the drawer from being dismissed by the escape key.

Accessibility

Keyboard

KeyAction
EscapeClose the drawer (unless isKeyboardDismissDisabled)
TabMove focus within the drawer (focus is trapped)

ARIA

  • role="dialog" with aria-modal="true" on the drawer panel (via RAC Modal)
  • Dialog accessible name via aria-label
  • aria-label="Close" on the built-in close button

Best Practices

  • Always provide an aria-label so the dialog has an accessible name.
  • Use isDismissable to allow closing via backdrop click.
  • Don't nest drawers.
