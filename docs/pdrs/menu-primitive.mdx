# Menu Primitive PDR

## Purpose

Headless ARIA menu for action menus, dropdowns, context menus. Built on React Aria's `useMenu` + `TreeState` (not `ListBox`).

**Usage:**
- Action menus: `onAction(key)`
- Selection menus: `selectionMode` + `selectedKeys`

**Architecture:**
- `Menu` — overlay-agnostic ARIA surface
- `MenuTrigger` — orchestrator coordinating trigger + overlay + menu via slots
- `SubmenuTrigger` — nested orchestrator for hierarchical menus

## Goals & Constraints

**Goals:**
- Headless ARIA menu built on React Aria / React Stately
- Controlled and uncontrolled selection/open state
- Minimal API: `Menu`, `MenuItem`, `MenuSection`, `MenuTrigger`, `SubmenuTrigger`

**Constraints:**
- File-local helpers only; React Aria as single source of behavior
- `Menu` does NOT own focus containment (managed by overlay primitives)
- v1: non-virtualized menus; no drag & drop
- Use `BentoError` from `@bento/error`

## Dependencies

**React Aria:** `useMenu`, `useMenuItem`, `useMenuSection`, `useMenuTrigger`, `useMenuTriggerState`, `TreeState`, `useTreeState`

**Bento:** `@bento/slots`, `@bento/use-props`, `@bento/use-data-attributes`, `@bento/error`, `@bento/container`, overlay primitives (for compositions)

## Menu vs ListBox

- **State**: `useMenu` uses `TreeState<T>` (hierarchical); `useListBox` uses `ListState<T>` (flat)
- **ARIA**: Menu uses `role="menu"` + `menuitem*`; ListBox uses `role="listbox"` + `option`
- **Structure**: Menus support submenus; ListBox is single-level

## Implementation Patterns

| Concept | Implementation |
|---------|----------------|
| State | `TreeState<T>` from `useTreeState` |
| Slots | `withSlots('BentoMenu', ...)` |
| Render props | `className`, `style`, `children` can be functions: `({ isSelected, isFocused, isHovered, isPressed, isDisabled }) => ...` |
| data-* | `useDataAttributes` (memoized) |
| Open state | `useMenuTriggerState` in `MenuTrigger` |
| Selection | React Aria `MultipleSelection` |

## Internal Structure

**Contexts:** `MenuStateContext` (`TreeState`), `MenuTriggerStateContext`, `RootMenuTriggerStateContext`
  - **Slots** distribute props; **Contexts** share state objects for React Aria hooks

**Hooks:** `useMenuDataAttributes`

## React Aria Integration

All behavior from React Aria hooks. Bento only adapts prop shapes, distributes via slots/context, adds controlled/uncontrolled plumbing.

## Public API

**Components** (generic `T extends object`):

- `Menu<T>` — root menu surface (`role="menu"`); requires `aria-label` or `aria-labelledby`
- `MenuItem<T>` — individual items; uses `id` prop for identification (React `key` for reconciliation)
  - Supports `href` for links (renders as `<a>`)
  - `textValue` required when children aren't plain string (for typeahead)
- `MenuSection<T>` — labeled groups
- `Separator` — visual divider (`<div role="separator">`)
- `MenuTrigger` — orchestrator; renders no DOM, distributes props via slots and provides context
- `SubmenuTrigger` — nested orchestrator; consumes `RootMenuTriggerStateContext`
  - Props: `delay` (hover timing)
  - Slots: `trigger` (parent item), `submenu` (nested `Menu`)

**Types/contexts:**

- `MenuProps<T>`, `MenuItemProps<T>`, `MenuSectionProps<T>`, `MenuTriggerProps`
- `MenuState<T>` — alias for `TreeState<T>`
- `MenuTriggerState` — re-export from React Stately
- Contexts: `MenuStateContext`, `MenuTriggerStateContext`, `RootMenuTriggerStateContext`
  - **Slots** distribute props (`menuTriggerProps`, `menuProps`) to children
  - **Contexts** share state objects that React Aria hooks consume (e.g., `useMenuItem(props, state, ref)`)
- `SelectionMode` — `'none' | 'single' | 'multiple'`

**Key Menu props:**
- `closeOnSelect` — closes after selection (default: `true` for single, `false` for multiple)
- `disabledKeys`, `shouldFocusWrap`

**Key MenuTrigger props:**
- `trigger` — `'press' | 'longPress'`; `isOpen`, `defaultOpen`, `onOpenChange`
- Focus strategy auto-set: keyboard → `'first'`

## Controlled vs Uncontrolled

**Selection:** Controlled (`selectedKeys` + `onSelectionChange`) or uncontrolled (`defaultSelectedKeys`)

**Open state:** Controlled (`isOpen` + `onOpenChange`) or uncontrolled (`defaultOpen`) on `MenuTrigger`

**onClose:** `Menu` receives `onClose` from `useMenu`; `MenuTrigger` wires to `state.close()` (automatic)

**Collection stability:** Use stable `items` references

## DOM Structure

```html
<div role="menu" aria-label="Actions">
  <div role="menuitem">New</div>
  <div role="menuitemcheckbox" aria-checked="true" data-checked="true">Autosave</div>
</div>
```

**Submenus:** Parent has `aria-haspopup="menu"`, `data-has-submenu="true"`; `SubmenuTrigger` manages hover/focus

## Form Integration

**Input-like:** `selectedKeys`/`onSelectionChange` as value; wrap with field component (`@bento/control`, `@bento/field-error`)

**Action:** `onAction(key)`

## Accessibility

- **Roles:** `role="menu"`, `menuitem*`, `role="group"`
- **Keyboard:** arrows, Home/End, typeahead, Escape; submenus via arrow right/left (respects RTL); Escape closes submenu → parent
- **Focus:** containment in overlay primitives; `data-focus-visible` for styling
- **Screen readers:** ARIA attributes from React Aria

## I18n, RTL, Mobile

- **RTL:** Keyboard nav/submenu arrows respect direction via `useLocale`
- **Mobile:** Pointer/touch normalized; `trigger="longPress"` for context menus
- **I18n:** All text/labels from consumers; typeahead uses locale-aware collators

## Performance

React Aria/Stately handle all behavior; data attributes memoized; v1 targets small, non-virtualized menus


## Data Attributes

**Menu container:**

| Attribute | Description | Values |
|-----------|-------------|--------|
| `data-empty` | No items | `"true"` / omitted |
| `data-focused` | Menu focused | `"true"` / omitted |
| `data-focus-visible` | Show focus ring | `"true"` / omitted |
| `data-selection-mode` | Selection mode | `"none"`, `"single"`, `"multiple"` |

**Menu item:**

| Attribute | Description | Values |
|-----------|-------------|--------|
| `data-selected` | Item selected | `"true"` / omitted |
| `data-checked` | Only when `role="menuitemcheckbox"` or `"menuitemradio"` | `"true"` / omitted |
| `data-disabled` | Disabled | `"true"` / omitted |
| `data-hovered` | Hovered | `"true"` / omitted |
| `data-focused` | Focused | `"true"` / omitted |
| `data-focus-visible` | Keyboard focus | `"true"` / omitted |
| `data-pressed` | Pressed | `"true"` / omitted |
| `data-has-submenu` | Opens submenu | `"true"` / omitted |
| `data-open` | Submenu open | `"true"` / omitted |

## Slot Map

`MenuTrigger` orchestrator slots:

| Slot | Description | Required |
|------|-------------|----------|
| `trigger` | Opens/closes menu (receives `menuTriggerProps`) | Yes |
| `overlay` | Overlay container (receives `isOpen` via `MenuTriggerStateContext`) | Yes |
| `menu` | Menu surface (receives `menuProps`) | Yes |

Note: Slots work through nested children. In the example, `Portal` wraps `Overlay slot="overlay"`—the slot propagates through React's children tree.

`Menu` wrapped in `withSlots` for composition (e.g. `<Menu slot="menu">`)

## Examples

### Basic + icons

```tsx
<Menu onAction={(key) => console.log(key)} aria-label="File">
  <MenuItem id="new">New</MenuItem>
  <MenuItem id="copy" textValue="Copy"><Icon name="copy" /> Copy</MenuItem>
</Menu>
```

### Selection + render props

```tsx
const [selected, setSelected] = useState(new Set(['autosave']));

<Menu selectionMode="single" selectedKeys={selected} onSelectionChange={setSelected} aria-label="Settings">
  <MenuItem id="autosave" className={({ isSelected }) => isSelected ? 'bg-blue' : ''}>Autosave</MenuItem>
</Menu>
```

### MenuTrigger

```tsx
<MenuTrigger>
  <Button slot="trigger">Actions</Button>
  <Portal>
    <Overlay slot="overlay">
      <Menu slot="menu" aria-label="Actions">
        <MenuItem id="edit">Edit</MenuItem>
      </Menu>
    </Overlay>
  </Portal>
</MenuTrigger>
```

### Links

```tsx
<MenuItem id="home" href="/">Home</MenuItem>
```

### Sections

```tsx
<Menu aria-label="Edit">
  <MenuSection title="Clipboard">
    <MenuItem id="cut">Cut</MenuItem>
    <MenuItem id="copy">Copy</MenuItem>
  </MenuSection>
  <Separator />
  <MenuSection title="History">
    <MenuItem id="undo">Undo</MenuItem>
  </MenuSection>
</Menu>
```

### Submenus

```tsx
<Menu aria-label="File">
  <SubmenuTrigger delay={200}>
    <MenuItem id="share" slot="trigger">Share →</MenuItem>
    <Menu slot="submenu" aria-label="Share">
      <MenuItem id="email">Email</MenuItem>
      <MenuItem id="slack">Slack</MenuItem>
    </Menu>
  </SubmenuTrigger>
</Menu>
```

## Architecture Summary

- **`Menu`** — Headless ARIA menu surface (like `ListBox`)
- **`MenuTrigger`** — Orchestrator composing trigger + overlay + menu (like `Select`)
- **`SubmenuTrigger`** — Nested orchestrator for hierarchical menus
