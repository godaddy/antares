# Text Primitive PDR

## Purpose

The Text Primitive will be the host for the lowest level text content within the component library, supporting many of the compositions expecting to display text content. This expects to include buttons, headlines, list items, among other elements.

## API

The API interface for the resource should include the following props:

```tsx
interface Text {
    /** @default null */
    align?: 'start' | 'center' | 'end' | 'justify';
    /** @default 'span' */
    as?: String;
    /** @default null */
    children?:  ReactNode;
    /** @default null */
    maxLines?: Number;
    /** @default false */
    visuallyHidden?: Boolean;
    /** @default null */
    wrap?: Boolean | 'balance' | 'pretty' | 'stable';
}
```

### `align`

The `align` prop is an analog to the CSS `text-align` property. When not provided, the alignment will inherit from ancestors eventually provided from the page level which typically defaults to `start`.

### `as`

The `as` prop determines the HTML element to render. If not provided, the default value will be `span` to represent the least semantic element. We may consider a dev warning message to consider something more semantic when a value is not provided. We also do not expect to provide strict HTML tag validation here, such that new HTML tags added over time will be immediately supported. The validity of the tag will be the responsibility of the author. Incoming `<Text as='marquee'/>`.

Some `as` props will produce [different but expected presentations](https://developer.mozilla.org/en-US/docs/Web/HTML/Reference/Elements/u#other_elements_to_consider_using) such as `<Text as='em/>` and `<Text as='code'/>`.

### `children`

The `children` prop places the content provided within the HTML element tags. While we expect this to primarily provided as a `String`, we allow a `ReactNode` for flexibility in use. This allows the component to be used to inherit font properties to larger compositions where appropriate. If `null` is provided, the component will not render. This helps with larger compositional layout needs, so phantom gaps do not appear when content is missing.

### `maxLines`

The `maxLines` prop would provide truncation to the text through additional CSS properties. Based on browser support, truncating by line using [`line-clamp`](https://developer.mozilla.org/en-US/docs/Web/CSS/line-clamp) may not be supported and will instead use [traditional CSS truncation techniques](https://css-tricks.com/snippets/css/truncate-string-with-ellipsis/).

### `visuallyHidden`

The `visuallyHidden` prop is a flag that indicates if the content is only meant to be discoverable by assistive technologies, specifically screen readers. This is provided as a configuration on the text component because content which is meant to be read only by a screen reader is most commonly text content. This also avoids the necessity to require a utility that adds this functionality to elements that are inappropriate. We can expect to use [modern visually hidden styles](https://github.com/WickyNilliams/cally/blob/main/src/utils/styles.ts#L18-L23) to support this functionality. However, we should also include the [`:not(:focus):not(:active)` recommendation](https://www.scottohara.me/blog/2017/04/14/inclusively-hidden.html) to ensure the content is not visible when the user is expected to find the text within the page.

> [!NOTE]
>
> I'm open to the possibility of avoiding this in favor of [the `<VisuallyHidden/>` component found within React Aria](https://github.com/adobe/react-spectrum/tree/main/packages/%40react-aria/visually-hidden) and can be later used to wrap `<Text/>` or for this to turn on its usage from within (less likely). My opinion is the implementation of this component is more complex that could otherwise be handled by CSS alone. Thouugh I do recognize the use of wrapping non-text elements in this, as is the case for [React Aria's `<Checkbox/>` component](https://github.com/adobe/react-spectrum/blob/main/packages/react-aria-components/src/Checkbox.tsx#L250-L252).

### `wrap`

Based on several hours of [research and exploration](https://blog.damato.design/posts/antidatalossconfigurationism/), my recommendation is to include a `wrap` prop that aligns to the options provided by `text-wrap` in CSS. This will support most use cases for wrapping. Except in the case where `white-space` is meant to be `pre`, specifically for `<pre/>` and `<kbd/>` elements. For these, if `wrap=true` is explicitly provided, we'll update the `white-space` to `pre-wrap` to support the use case. Additionally, we will default all text components to include `overflow-wrap: break-word` to support long words and URLs as a baseline.

## React Aria/Spectrum

React Aria does not have a Text primitive. The component within React Spectrum offers very little in terms of [their API](https://github.com/adobe/react-spectrum/tree/main/packages/%40react-types/text/src). It seems they expose `children` and `slot` as props for most variations of the component. A slightly unique variation is `Heading` which can also accept a `level` prop. The expectation is to leverage the current infrastructure of Bento to introduce `slots` along with other overrides available in the Bento project.

[The docs page](https://react-spectrum.adobe.com/react-spectrum/Text.html#text) presents additional configuration options for the component which seem to be more layout related than text related. These are most likely inherited from a base component as we expect to inherit from our own `useRenderProps` from within Bento.

## Code examples

### Basic usage

```tsx
/* Renders a span element with the text "Hello, world!" */
<Text>Hello, world!</Text>
```

### No children

```tsx
/* Renders nothing */
<Text></Text>
```

### Truncation

```tsx
/* Renders a p element with the text "This is a long text that will be truncated after two lines." */
<Text as="p" maxLines={2}>This is a long text that will be truncated after two lines.</Text>
```

### Visually hidden

```tsx
/* Renders a span element with the text "This content is only visible to screen readers." */
<Text visuallyHidden>This content is only visible to screen readers.</Text>
```

### Text alignment

```tsx
/* Renders a h1 element with the text "This text will be centered." */
<Text as="h1" align="center">This text will be centered.</Text>
```

### Text wrapping

```tsx
/* Renders a h2 element with the text "This text will wrap with character balance to the next line." */
<Text as="h2" wrap="balance">This text will wrap with character balance to the next line.</Text>
```
