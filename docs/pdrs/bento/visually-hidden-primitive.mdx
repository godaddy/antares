# VisuallyHidden Primitive PDR

## Purpose

The VisuallyHidden primitive provides content that is accessible to screen readers and other assistive technologies while being visually hidden from sighted users. This is a fundamental accessibility primitive that enables developers to provide additional context, instructions, or navigation aids without cluttering the visual interface.

This primitive is essential for implementing accessible patterns like skip links, screen reader-only form labels, additional context for interactive elements, and descriptive text that supplements visual cues. It serves as a building block for other accessible components throughout the Bento ecosystem and is consumed by form components, navigation systems, and any component requiring accessible hidden content.

### Unique Attributes

The VisuallyHidden primitive is unique in its simplicity and critical importance:

- **React Aria foundation**: Built directly on React Aria's proven VisuallyHidden implementation
- **Zero visual interference**: Uses battle-tested CSS techniques to hide content visually while maintaining accessibility
- **Screen reader preservation**: Ensures content remains available to assistive technologies
- **Universal applicability**: Can be used with any content type (text, interactive elements, etc.)
- **Slot-based customization**: Adds Bento's flexible customization system on top of React Aria's solid foundation
- **Performance optimized**: Minimal runtime overhead with CSS-based hiding

## Primitive Composition

The VisuallyHidden primitive is built using:

1. **React Aria's VisuallyHidden** - The core foundation providing proven accessibility behavior
2. **withSlots** and **useProps** - For slot-based composition and customization following Bento patterns

```tsx
import { VisuallyHidden as AriaVisuallyHidden } from 'react-aria';
import { withSlots } from '@bento/slots';
import { useProps } from '@bento/use-props';

export const VisuallyHidden = withSlots(
  'BentoVisuallyHidden',
  function VisuallyHidden(args) {
    const { props, apply } = useProps(args);
    const { children, as, elementType, ...rest } = props;

    // Support both `as` (community standard) and `elementType` (React Aria)
    const element = as || elementType;

    return (
      <AriaVisuallyHidden
        {...apply(rest)}
        elementType={element}
      >
        {children}
      </AriaVisuallyHidden>
    );
  }
);
```

This is a **UI primitive** that renders HTML elements with specific styling to achieve visual hiding while maintaining accessibility.

## Internal Structure & Reuse Potential

This is already the minimal reusable unit for visual hiding functionality. The primitive:

- **Highly reusable**: Used across form components, navigation, buttons, and accessibility patterns
- **Atomic responsibility**: Only handles visual hiding, delegates to React Aria for implementation
- **Foundation for patterns**: Skip links, screen reader labels, accessible descriptions, form enhancements
- **Shared behavior**: The React Aria foundation ensures consistent behavior across all Bento components

The component is intentionally simple and focused, making it perfect for composition in larger components without introducing complexity. It serves as a foundational building block that other Bento components can rely on for accessible hidden content.

## React Aria or External Hook Integration

- **React Aria's VisuallyHidden**: Uses their complete implementation with proven CSS hiding techniques
- **Full compatibility**: Maintains all React Aria's accessibility features and browser compatibility
- **No gaps in coverage**: React Aria provides complete coverage for this use case

React Aria's implementation handles all edge cases, cross-browser concerns, and accessibility requirements, so no additional logic is required.

## Architecture & Features

- **Custom elements**: Supports `as` prop (community standard) and `elementType` (React Aria compatibility) for semantic HTML elements
- **Children support**: Accepts any React children content
- **React Aria foundation**: Uses React Aria's battle-tested hiding implementation
- **No custom CSS**: Relies entirely on React Aria's proven hiding technique
- **Slot composition**: Adds Bento's customization layer on top of React Aria's foundation

- **Cross-browser compatibility**: React Aria handles browser inconsistencies

- **React Aria compatibility**: Inherits React Aria's international and mobile support

## Competitive Research

### React Aria
- **Implementation**: Uses proven CSS clipping technique with `position: absolute` and `clip: rect(0, 0, 0, 0)`
- **API**: Simple `<VisuallyHidden elementType="span">` interface
- **Customization**: Limited to element type and standard HTML props
- **Bundle**: Part of larger React Aria ecosystem
- **Accessibility**: Thoroughly tested across screen readers and browsers
- **Props**:
  ```tsx
  interface VisuallyHiddenProps {
    children: React.ReactNode;
    elementType?: keyof JSX.IntrinsicElements; // default: 'span'
    // + all standard HTML props
  }
  ```

### Reach UI
- **Implementation**: Similar CSS approach but with slightly different clipping values
- **API**: `<VisuallyHidden>` with children only
- **Customization**: No element type customization, always renders as `<span>`
- **Bundle**: Standalone package, smaller footprint
- **Accessibility**: Good but less comprehensive testing than React Aria
- **Props**:
  ```tsx
  interface VisuallyHiddenProps {
    children: React.ReactNode;
    // + standard HTML span props only
  }
  ```

### Chakra UI
- **Implementation**: Uses `position: absolute` with `height: 1px` and `overflow: hidden`
- **API**: `<VisuallyHidden>` with additional styling props support
- **Customization**: Supports Chakra's styling system (sx, etc.)
- **Bundle**: Integrated with Chakra's theming system
- **Accessibility**: Solid implementation but focused on Chakra ecosystem
- **Props**:
  ```tsx
  interface VisuallyHiddenProps extends HTMLChakraProps<'span'> {
    children: React.ReactNode;
    as?: keyof JSX.IntrinsicElements; // Chakra's polymorphic prop
    // + Chakra styling props (sx, color, etc.)
  }
  ```

### Headless UI
- **Implementation**: Doesn't provide this primitive directly
- **Approach**: Expects developers to implement their own or use CSS utilities
- **Alternative**: Recommends using Tailwind's `sr-only` utility class
- **Philosophy**: Focuses on interactive components, not utility primitives
- **Props**: N/A - No component provided

### Radix UI
- **Implementation**: CSS-based hiding with `position: absolute` and clipping
- **API**: `<VisuallyHidden>` with `asChild` prop for composition
- **Customization**: Supports `asChild` for polymorphic rendering
- **Bundle**: Part of Radix primitives collection
- **Accessibility**: Well-tested implementation
- **Props**:
  ```tsx
  interface VisuallyHiddenProps {
    children: React.ReactNode;
    asChild?: boolean; // renders as child element instead of span
    // + standard HTML props
  }
  ```

### Bento's Approach
- **Implementation**: Built directly on React Aria's proven foundation
- **API**: Combines React Aria's reliability with Bento's slot system
- **Customization**: Slot-based overrides, render props, and React Aria's element types
- **Bundle**: Minimal wrapper around React Aria
- **Accessibility**: Inherits React Aria's comprehensive testing plus Bento's patterns
- **Unique**: Only implementation that combines React Aria's reliability with advanced slot-based customization
- **Props**:
  ```tsx
  interface VisuallyHiddenProps {
    children: React.ReactNode;
    as?: keyof JSX.IntrinsicElements; // Community standard (preferred)
    elementType?: keyof JSX.IntrinsicElements; // React Aria compatibility
    className?: string | ((data: RenderPropData) => string); // render prop
    style?: CSSProperties | ((data: RenderPropData) => CSSProperties); // render prop
    // + all standard HTML props
  }
  ```

### Key Differences Summary

| Feature | React Aria | Reach UI | Chakra UI | Radix UI | Bento |
|---------|------------|----------|-----------|----------|--------|
| CSS Technique | Clipping | Clipping | Height + Overflow | Clipping | React Aria's |
| Element Types | ✅ | ❌ | ❌ | ❌ (uses asChild) | ✅ |
| Customization | Basic | None | Styling system | asChild | Slots + Render Props |
| Accessibility | Excellent | Good | Good | Good | Excellent (React Aria) |
| Ecosystem | React Aria | Standalone | Chakra | Radix | Bento |
| Props API | Simple | Minimal | Styling-rich | Polymorphic | Comprehensive |

**Bento's Advantage**: Combines React Aria's battle-tested accessibility implementation with Bento's flexible slot-based customization system, providing the most reliable foundation with the most customization options. The props API is the most comprehensive, supporting everything from basic usage to advanced slot-based overrides.
