# Select Primitive PDR

## Purpose

The Select primitive provides an accessible, customizable dropdown that overcomes native `<select>` limitations—enabling custom option rendering, grouped sections, and rich UX patterns while remaining headless. Follows [W3C ARIA combobox pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/examples/combobox-select-only/).

### Unique Attributes

- **Compound components** (`Select`, `SelectTrigger`, `SelectValue`, `SelectContent`, `SelectOption`, `SelectGroup`, `SelectLabel`): Enables flexible composition vs monolithic component that would limit customization
- **Builds on ListBox**: Reuses proven selection patterns rather than duplicating accessibility/keyboard logic
- **Portal rendering**: Solves [dropdown clipping issues](https://github.com/floating-ui/floating-ui/discussions/1965) common with overflow:hidden containers
- **React Aria positioning**: Uses [`useOverlayPosition`](https://react-spectrum.adobe.com/react-aria/useOverlayPosition.html) for smart positioning with flip/shift collision detection and RTL support, integrated with React Aria's accessibility system
- **Hidden native `<select>`**: Maintains form submission and [autofill](https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/autocomplete) compatibility
- **Custom value rendering**: Allows different display format than option (e.g., "USA" in dropdown, "🇺🇸 United States" when selected)
- **Typeahead navigation**: Essential UX for long lists without complexity of full search

## Primitive Composition

**Components**: `Select` (root with context), `SelectTrigger` (button using triggerProps from useSelect), `SelectValue`, `SelectContent` (portal + React Aria positioning + Bento's ListBox), `SelectOption` (wrapper around ListBoxItem, maps value→key), `SelectGroup` (alias for ListBoxSection), `SelectLabel`

**Hooks**: [`useSelect`](https://react-spectrum.adobe.com/react-aria/useSelect.html) (includes useMenuTrigger internally), [`useSelectState`](https://react-spectrum.adobe.com/react-stately/useSelectState.html), [`useOverlay`](https://react-spectrum.adobe.com/react-aria/useOverlay.html), [`useOverlayPosition`](https://react-spectrum.adobe.com/react-aria/useOverlayPosition.html), `useProps`, `useDataAttributes`, `withSlots`

This is a **UI component** that combines behavioral hooks with compound component composition.

## Implementation Pattern

Each component uses withSlots, useProps, and useDataAttributes:

```tsx
import { useSelect, HiddenSelect, useOverlay, useOverlayPosition, mergeProps } from 'react-aria';
import { useSelectState } from 'react-stately';
import { useRef } from 'react';
import { ListBox, ListBoxItem, ListBoxSection, ListStateContext } from '@bento/listbox';
import { createPortal } from 'react-dom';
import { useDataAttributes } from '@bento/use-data-attributes';
import { useProps } from '@bento/use-props';
import { withSlots } from '@bento/slots';

// Select - Root component that provides context and renders HiddenSelect
export const Select = withSlots('BentoSelect', function Select(args) {
  const { props, apply } = useProps(args);
  const state = useSelectState(props);
  const triggerRef = useRef<HTMLButtonElement>(null);
  const popoverRef = useRef<HTMLDivElement>(null);
  
  const {
    triggerProps,
    valueProps,
    menuProps,
    labelProps,
    descriptionProps,
    errorMessageProps,
    hiddenSelectProps
  } = useSelect(props, state, triggerRef);
  
  const contextValue = {
    state,
    triggerRef,
    popoverRef,
    triggerProps,
    valueProps,
    menuProps,
    labelProps,
    descriptionProps,
    errorMessageProps
  };
  
  return (
    <SelectContext.Provider value={contextValue}>
      <ListStateContext.Provider value={state}>
        {props.children}
      </ListStateContext.Provider>
      {props.name && (
        <HiddenSelect
          state={state}
          triggerRef={triggerRef}
          {...hiddenSelectProps}
        />
      )}
    </SelectContext.Provider>
  );
});

// SelectTrigger - Button that opens dropdown
// triggerProps from useSelect already includes button behavior via useMenuTrigger
export const SelectTrigger = withSlots('BentoSelectTrigger', function SelectTrigger(args) {
  const { props, apply } = useProps(args);
  const { state, triggerRef, triggerProps } = useSelectContext();
  
  return (
    <button
      ref={triggerRef}
      {...apply(triggerProps, ['children'])}
      {...useDataAttributes({ open: state.isOpen, disabled: state.isDisabled })}
    >
      {props.children}
    </button>
  );
});

// SelectValue - Displays the selected value or placeholder
export const SelectValue = withSlots('BentoSelectValue', function SelectValue(args) {
  const { props } = useProps(args);
  const { state, valueProps } = useSelectContext();
  const selectedItem = state.selectedKey != null ? state.collection.getItem(state.selectedKey) : null;
  
  return (
    <span {...valueProps}>
      {selectedItem ? selectedItem.textValue : props.placeholder}
    </span>
  );
});

// SelectContent - Portal-rendered dropdown using Bento's ListBox
export const SelectContent = withSlots('BentoSelectContent', function SelectContent(args) {
  const { props, apply } = useProps(args);
  const { state, popoverRef, triggerRef, menuProps } = useSelectContext();
  const { overlayProps } = useOverlay({ isOpen: state.isOpen, onClose: () => state.close() }, popoverRef);
  const { overlayProps: positionProps } = useOverlayPosition({
    targetRef: triggerRef,
    overlayRef: popoverRef,
    placement: 'bottom start',
    offset: 8
  });
  
  if (!state.isOpen) return null;
  
  // Bento's ListBox consumes ListStateContext provided by Select root
  // menuProps from useSelect provide ARIA attributes for the listbox
  return createPortal(
    <div 
      ref={popoverRef} 
      {...apply(mergeProps(overlayProps, positionProps), ['children'])} 
      {...useDataAttributes({ open: true })}
    >
      <ListBox {...menuProps}>
        {props.children}
      </ListBox>
    </div>,
    document.body
  );
});

// SelectOption - Wrapper around ListBoxItem that maps value prop to key
export const SelectOption = withSlots('BentoSelectOption', function SelectOption(args) {
  const { props } = useProps(args);
  // Map value prop (Select API) to key prop (ListBoxItem/React Aria API)
  const { value, ...restProps } = props;
  
  return <ListBoxItem key={value} {...restProps} />;
});

// SelectGroup - Reuses Bento's ListBoxSection
export const SelectGroup = ListBoxSection;

// SelectLabel - Reuses ListBoxSection.Label pattern
export const SelectLabel = ListBoxSection.Label;
```

> **Note:** This is a simplified example. For the full implementation including dismiss and focus handling, see the [Portal & Overlay Strategy](#portal--overlay-strategy) section.

**Context provides:** `state` (useSelectState extends ListState), `triggerRef`, `popoverRef`, `menuProps`, `triggerProps`, `valueProps`, ARIA props  
**Hooks used:** useSelect, useSelectState, useOverlay, useOverlayPosition  
**Components reused:** 
- `SelectContent` renders Bento's `ListBox` component with `menuProps` from `useSelect`
- `SelectOption` = thin wrapper around `ListBoxItem` that maps `value` prop → `key` prop for more intuitive Select API
- `SelectGroup` = `ListBoxSection` (alias)  
- `SelectLabel` = `ListBoxSection.Label` (alias)

**See:** ListBox implementation for detailed option/group patterns

## Internal Structure & Reuse Potential

**Component reuse strategy**:
- **SelectContent**: Renders Bento's `ListBox` component internally, passing `menuProps` from `useSelect`. The Select root provides `ListStateContext` (from `useSelectState` which extends `ListState`), which `ListBox` consumes automatically.
- **SelectOption**: Thin wrapper around `ListBoxItem` that maps the `value` prop (intuitive for Select API) to the `key` prop (required by ListBoxItem/React Aria collection system). This provides a better API surface while reusing existing primitives.
- **SelectGroup, SelectLabel**: Direct aliases around `ListBoxSection` and `ListBoxSection.Label` respectively. This maximizes reuse of existing, tested primitives.
- **SelectContext**: Provides Select-specific context (triggerRef, popoverRef, menuProps) while `ListStateContext` (provided by Select root) handles shared list state.

**Future reuse potential**:
- Select trigger/value patterns reusable in Combobox, Autocomplete, DatePicker
- Overlay positioning pattern reusable in Popover, Menu, etc.

## React Aria & External Integration

**React Aria provides core behavior** ([`useSelect`](https://react-spectrum.adobe.com/react-aria/useSelect.html) (which internally uses `useMenuTrigger`), [`useSelectState`](https://react-spectrum.adobe.com/react-stately/useSelectState.html))—handling ARIA attributes, keyboard nav, and state. We extend with:

- **Slots/render props**: Enable deep customization without forking
- **Data attributes**: Expose state for CSS (avoiding [ARIA for styling](https://www.w3.org/WAI/ARIA/apg/practices/read-me-first/#do-not-use-aria-to-style-content))
- **Positioning**: [`useOverlayPosition`](https://react-spectrum.adobe.com/react-aria/useOverlayPosition.html) provides smart positioning with flip/shift collision detection, RTL support, and viewport boundary handling, fully integrated with React Aria's accessibility system

### Portal & Overlay Strategy

`SelectContent` uses React Aria's overlay system with native portals:

- **Positioning**: [`useOverlayPosition`](https://react-spectrum.adobe.com/react-aria/useOverlayPosition.html) handles smart positioning with flip/shift collision detection, RTL support, and viewport boundaries
- **Overlay behavior**: `useOverlay` handles dismiss, focus trap, and accessibility
- **Portal rendering**: Native `createPortal` to `document.body` prevents clipping by overflow containers

```tsx
import { useOverlay, useOverlayPosition, mergeProps } from 'react-aria';
import { createPortal } from 'react-dom';

const { overlayProps } = useOverlay({ isOpen: state.isOpen, onClose: state.close }, overlayRef);
const { overlayProps: positionProps } = useOverlayPosition({
  targetRef: triggerRef,
  overlayRef,
  placement: 'bottom start',
  offset: 8
});

// Merge React Aria hook props together
const mergedAriaProps = mergeProps(overlayProps, positionProps);

return createPortal(
  <div ref={overlayRef} {...mergedAriaProps}>
    {children}
  </div>,
  document.body
);
```

When integrating with Bento's `useProps` and `apply()` to support user props and slot customization:

```tsx
import { useOverlay, useOverlayPosition, mergeProps } from 'react-aria';
import { createPortal } from 'react-dom';
import { useProps } from '@bento/use-props';

const { props, apply } = useProps(args);
const { overlayProps } = useOverlay({ isOpen: state.isOpen, onClose: state.close }, overlayRef);
const { overlayProps: positionProps } = useOverlayPosition({
  targetRef: triggerRef,
  overlayRef,
  placement: 'bottom start',
  offset: 8
});

// Merge React Aria hook props, then apply with user props via Bento's apply()
const mergedAriaProps = mergeProps(overlayProps, positionProps);

return createPortal(
  <div 
    ref={overlayRef} 
    {...apply(mergedAriaProps, ['children'])}
    {...useDataAttributes({ open: true })}
  >
    {props.children}
  </div>,
  document.body
);
```

## Architecture & Features

### Component Structure

```tsx
import { 
  Select, 
  SelectTrigger, 
  SelectValue, 
  SelectContent, 
  SelectOption,
  SelectGroup,
  SelectLabel 
} from '@bento/select';

<Select>
  <SelectTrigger>
    <SelectValue placeholder='Choose an option' />
  </SelectTrigger>
  <SelectContent>
    <SelectOption value='1'>Option 1</SelectOption>
    <SelectGroup>
      <SelectLabel>Group Label</SelectLabel>
      <SelectOption value='2'>Option 2</SelectOption>
    </SelectGroup>
  </SelectContent>
</Select>
```

### Rendered DOM Structure

```html
<!-- Trigger (follows ARIA combobox pattern) -->
<button
  role="combobox"
  aria-expanded="false"
  aria-haspopup="listbox"
  data-open="false"
>
  <span>Selected Value</span>
  <svg><!-- chevron icon --></svg>
</button>

<!-- Hidden native select for forms -->
<select aria-hidden="true" tabindex="-1" style="position: absolute; ...">
  <option value="1">Option 1</option>
  <option value="2">Option 2</option>
</select>

<!-- Portal-rendered dropdown -->
<div style="position: fixed; ...">
  <div role="listbox" aria-label="Select options">
    <div role="option" aria-selected="true" data-selected="true">Option 1</div>
    <div role="group" aria-labelledby="group-1">
      <span id="group-1">Group Label</span>
      <div role="option" aria-selected="false">Option 2</div>
    </div>
  </div>
</div>
```

### Key Props & State

| Prop                              | Type              | Purpose                                     |
| --------------------------------- | ----------------- | ------------------------------------------- |
| `value`/`defaultValue`            | `string`          | Controlled/uncontrolled selection           |
| `open`/`defaultOpen`              | `boolean`         | Controlled/uncontrolled dropdown visibility |
| `onValueChange`                   | `(value) => void` | Selection callback                          |
| `onOpenChange`                    | `(open) => void`  | Visibility callback                         |
| `disabled`, `required`, `invalid` | `boolean`         | Form states                                 |
| `placeholder`                     | `string`          | Empty state text                            |
| `name`                            | `string`          | Form submission field name                  |
| `items`                           | `Iterable<T>`     | Dynamic collections                         |
| `disabledKeys`                    | `Set<string>`     | Disable specific options                    |

**State management**: Supports controlled/uncontrolled modes, syncs with hidden `<select>` for forms, manages focus and typeahead buffer.

### Form Integration

Uses React Aria's `HiddenSelect` component for native form compatibility. As shown in the [Implementation Pattern](#implementation-pattern) section, `useSelect` returns `hiddenSelectProps` which are passed to `HiddenSelect`. The hidden select:
- Renders conditionally when `name` prop is provided (enables form submission)
- Syncs automatically with select state
- Handles native form submission and browser autofill
- Maintains HTML form compatibility for server-side form handling

## Accessibility & Platform Support

**ARIA** ([combobox pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/#keyboard-interaction-12)):

- Roles: `combobox`, `listbox`, `option`, `group`
- Attributes: `aria-expanded`, `aria-selected`, `aria-invalid`, `aria-required`
- Keyboard: Arrow nav, `Enter`/`Space` select, `Escape` close, typeahead
- Focus: [Roving tabindex](https://www.w3.org/WAI/ARIA/apg/practices/keyboard-interface/#kbd_roving_tabindex), returns to trigger, `data-focus-visible`

**Platform**:

- **RTL**: [Auto-flip positioning](https://react-spectrum.adobe.com/react-aria/internationalization.html#rtl) and icons
- **Mobile**: [44px tap targets](https://www.w3.org/WAI/WCAG22/Understanding/target-size-minimum.html), scroll locking
- **I18n**: Localizable strings, locale-aware typeahead

## Customization

**Data attributes** for CSS ([not ARIA](https://www.w3.org/WAI/ARIA/apg/practices/read-me-first/#do-not-use-aria-to-style-content)):

- Trigger: `data-open`, `data-disabled`, `data-invalid`, `data-required`, `data-placeholder`
- Option: `data-selected`, `data-disabled`, `data-focused`
- Interactive: `data-focus-visible`, `data-hovered`, `data-pressed`

**Slots** for composition:

- `trigger`, `trigger.value`, `trigger.icon` - Button customization
- `content`, `content.viewport` - Dropdown container
- `option`, `option.label`, `option.icon` - Option rendering
- `group`, `group.label` - Grouped sections

## Prior Art

- **[React Aria](https://react-spectrum.adobe.com/react-aria/Select.html)**: Our accessibility foundation (hooks only, no components)
- **[Radix UI](https://www.radix-ui.com/primitives/docs/components/select)**: Good compound pattern (but limited slot customization)
- **[Headless UI](https://headlessui.com/react/listbox)**: Clean API (but rigid structure)
- **[Arco](https://arco.design/react/en-US/components/select)/[Ant](https://ant.design/components/select)**: Virtual scrolling, search (future scope)

**Why Bento**: Combines React Aria's accessibility with deep customization via slots/render props—avoiding the "all or nothing" approach of other libraries.

## Usage Examples

```tsx
import { 
  Select, 
  SelectTrigger, 
  SelectValue, 
  SelectContent, 
  SelectOption,
  SelectGroup,
  SelectLabel 
} from '@bento/select';

// Basic controlled select with groups
const [value, setValue] = useState('apple');
<Select value={value} onValueChange={setValue} name="fruit" required>
  <SelectTrigger>
    <SelectValue placeholder="Select a fruit" />
  </SelectTrigger>
  <SelectContent>
    <SelectGroup>
      <SelectLabel>Fruits</SelectLabel>
      <SelectOption value="apple">Apple</SelectOption>
      <SelectOption value="orange" disabled>Orange (Out of Stock)</SelectOption>
    </SelectGroup>
  </SelectContent>
</Select>

// Customization via slots and render props
<Select
  slots={{
    'option': ({ props, children, state }) => (
      <div {...props}>
        {state.isSelected && <CheckIcon />}
        {children}
      </div>
    )
  }}
>
  <SelectContent>
    <SelectOption
      value="custom"
      className={({ isFocused }) => isFocused ? 'focused' : ''}
    >
      Custom Option
    </SelectOption>
  </SelectContent>
</Select>

// Dynamic items API
const items = [{ id: 'apple', name: 'Apple' }, { id: 'orange', name: 'Orange' }];
<Select items={items}>
  {item => <SelectOption key={item.id}>{item.name}</SelectOption>}
</Select>
```

## Implementation Rationale

**Architecture decisions**:

- **Compound components**: Maximum flexibility vs monolithic API that limits customization
- **Build on ListBox**: Reuse [ARIA listbox](https://www.w3.org/WAI/ARIA/apg/patterns/listbox/) patterns rather than duplicate
- **Portal rendering**: Solve [overflow clipping](https://github.com/floating-ui/floating-ui/discussions/1965) and z-index issues
- **Hidden `<select>`**: Maintain form submission and [autofill](https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/autocomplete) compatibility
- **React Aria positioning**: Use [`useOverlayPosition`](https://react-spectrum.adobe.com/react-aria/useOverlayPosition.html) for consistency with existing Bento primitives (e.g., Drawer), seamless integration with React Aria's accessibility hooks, and proven positioning logic that handles edge cases (RTL, viewport boundaries, visual viewport on mobile)

**Future optimizations** (large lists):

- Virtual scrolling via [react-window](https://react-window.vercel.app/)
- Debounced typeahead
- Lazy section loading
- [React.memo](https://react.dev/reference/react/memo) for options

**Testing focus**: Keyboard navigation, screen readers, RTL, form submission, positioning edge cases.
