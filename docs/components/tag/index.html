Accessibility

Screen readers

Tags are <span> elements with no special ARIA role. Screen readers announce the tag's text as part of normal page content — no extra markup is needed.

Decorative icons

Icons passed as children must include aria-hidden="true" so they are skipped by screen readers. The text label is the sole carrier of meaning.

Indicator dot

The colored dot is decorative and invisible to screen readers. The text label must convey the status on its own — never rely on the dot alone to communicate meaning.

Best Practices

  • Use the indicator style in dense layouts like tables and dashboards. Use the filled style when the tag is a prominent, standalone element on a card or hero section.
  • Default to the small size in compact layouts like tables and lists. Use medium or large only when the tag needs to stand out as a primary piece of information.
  • Use the inline design only within running prose. In grouped or standalone contexts, the filled style provides better visual separation.
  • Do not stack multiple tags of different emphasis on the same item. Choose the most important one — too many competing labels lose their meaning.

When to Use

  • Showing the status of an item on a card, table row, or list entry (e.g. "Active", "Expired", "Pending")
  • Categorizing or grouping content so users can scan and filter quickly (e.g. "New", "Sale", "Beta")
  • Surfacing metadata about an item (e.g. "Pro", "Free trial", "Internal")
  • Communicating priority or category in project management and collaboration tools (e.g. "High priority", "In review")

When Not to Use

  • If users need to click or dismiss it — use a Button instead.
  • If the label is part of a sentence with no visual distinction — just use text.

Select

Select is a dropdown for picking one or more values from a list of options. It renders a labeled trigger and a popover listbox, supports single and multiple selection, and submits naturally inside a form.

TextField

TextField is a single-line or multiline text input with optional label, description, and error message. Use it in forms for short answers (email, search, name) or longer content (comments). It supports optional leading and trailing text adornments (e.g. currency or units).
