Field primitives

Composable primitives (Field, FieldLabel, FieldGroup, FieldError) for building form fields

Composition primitives. The field components (TextField, NumberField, DateField, Select, RadioGroup, CheckboxGroup) compose these internally — reach for those first. These primitives are also the building blocks for custom composite fields, e.g. an input and a select sharing one bordered box.

The field decoration is split into granular, composable pieces so the same structure is shared across every field:

  • Field — stacks a field's label, control, description, and error in one consistently spaced column (pure layout — a Flex column with the standard field gap).
  • FieldLabel — names the field and flags whether it's required (wraps React Aria Label, rendering an optional * indicator).
  • FieldGroup — the bordered box that visually unites a field's controls and reflects their shared focus, disabled, and invalid state. Used only by "boxed" fields (TextField, NumberField, Select, DateField); presentationally it is a React Aria Group.
  • FieldInput / FieldTextArea — the control the user types into (normalized React Aria Input/TextArea given antares spacing via Box).
  • FieldButton — an actionable control placed inside the box, such as the NumberField steppers or a search submit (a click target within the group).
  • FieldDescription — helper text that explains the field; renders nothing when empty (wraps the antares Text with slot="description").
  • FieldError — surfaces the field's validation messages when it is invalid, and nothing when valid (a thin wrapper over React Aria FieldError).

FieldLabel, FieldDescription, and FieldError each render null when they have no content, so callers pass label / description / errorMessage straight through without guarding.

Composition

A boxed field composes the primitives inside its React Aria field wrapper. Use Field as={...} to merge the field wrapper and the layout column into a single element:

A field without a box (RadioGroup / CheckboxGroup) composes Field + FieldLabel + FieldError and omits FieldGroup.

FieldGroup is purely presentational — it renders the styled Group and holds no React Aria context. The surrounding field wires value, disabled, and validation state into its controls.

Positioning controls inside the group

FieldGroup rounds its outer corners from DOM order: the group's CSS rounds the start corners of its first child and the end corners of its last child, so controls are seamless inside and rounded only at the box edges. Author children in visual order and the rounding follows — no per-control positioning prop. A lone control (a TextField input, a Select trigger) is both first and last, so it rounds both edges. The rules are RTL-safe (they use logical start/end corners).

States

The states disabled, readonly, and invalid are applied by the field wrapper (e.g. TextField, NumberField, Select). The group reflects them via data attributes on the group.

StateCSS Selector
disabled — Applied when the field wrapper is disabled.[data-disabled]
invalid — Applied when the field wrapper is invalid (e.g. validation failed).[data-invalid]
focus — Whether an element within the group has keyboard focus (for focus-ring styling).[data-focused]
