***REMOVED***Render Props

The `@bento/render-props` package provides a hook that unifies the component
and slot based props into a single interace. Both the component and slot-based
props can be specified as a `renderProp`. A `renderProp` is a function that
returns the value that should be applied as props to the component.

#***REMOVED***Installation

```shell
npm install --save @bento/render-props
```

#***REMOVED***useRenderProps

The package exposes a `useRenderProps` hook that can be used to apply the
render props to the component. The hook is designed to be used in conjunction
with the `@bento/slots` package as it allows the `slots` props that are
introduced on the component to override the props that are specified on the
component.

```javascript
import { useRenderProps } from '@bento/render-props';
import { useState } from 'react';

function Component(args) {
  const [value, setValue] = useState(0);
  const [props, apply] = useRenderProps(args, { value });
}
```

As seen from the example above, the `useRenderProps` hook takes two
arguments:

<ArgTypes of={Stories.useRenderProps} />

The `useRenderProps` hook response is modeled after React's `useState` hook and
returns an `array` with exactly two values that can be destructured:

<ArgTypes of={Stories.hook} />

In the examples we're following the naming convention of `props` and `apply`
for the two values that are returned. You can use any naming convention
for the destructured values.

##***REMOVED***props

The first item of the array is a `props`-like object. To be more specific, it's
a `Proxy` instance. This object provides the familiar `props` interface for
retrieving the assigned props of your component. As components can be altered by
the `slots` prop to introduce addition props or overrides to this Proxy, it will
automatically and transparently return the correct value.

When the prop that you're trying to access is specified as a `renderProp`, the
`Proxy` will automatically call the function and return the value returned by
the function. It's designed to do all the work for you so you can focus on
writing your components.

Just like normal `props`, you can also use this instance to spread the props
on your components:

```javascript
import { useRenderProps } from '@bento/render-props';
import { withSlots } from '@bento/slots';
import { useState } from 'react';

const Anchor = withSlots('Anchor', function AnchorComponent(args) {
  const [value, setValue] = useState(0);
  const [props, apply] = useRenderProps(args, { value });

  return <a {...props} />;
})
```

The primary use case for the `props` item is to interact with properties that
do not have a default value assigned to them or when you need to spread unknown
props to your component. When you interact with a prop that is set as renderProp
it will automatically call the function and return the value that is returned.
Given that no default value is applied, this renderProp will not be called
with the `original` value assigned.

```javascript
import { useRenderProps } from '@bento/render-props';
import { withSlots } from '@bento/slots';

const Anchor = withSlots('Anchor', function AnchorComponent(args) {
  const [props, apply] = useRenderProps(args);
  const { design, ...rest } = props;

  const className = design === 'foo' ? 'foo' : 'bar';
  return <a {...props} {...apply({ className })} />;
}

// Illustrative purposes only, useCallback is required in a real application
<Anchor design={function renderProp({ original, state }) {
  console.log(original);  // undefined - Design prop was only used to lookup values
  console.log(state);     // {} - No state was specified in the useRenderProps

  return 'foo';
}} />
```

##***REMOVED***apply

Unlike the `prop` item, the `apply` function is used to supply "default"
values to the props.

For example, you want to apply a class name to your component, instead of
assigning this directly to the component using `className="my-class"` you would
use the `apply` function and pass it `className` as key, and `my-class` as value.

```javascript
<Component className={({ original } => [original, 'my-class'].filter(Boolean).join(' ') )} />
```

Applying default values isn't the only use case; any property you 
pass to a component should be passed through the `apply` function. This ensures
that the `slots` prop can override the value passed to the component.
