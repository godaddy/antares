## Features

- In-flow collapsible panel built on RAC Disclosure
- Animated by default; `animate={false}` disables the transition
- `minSize` "peek": the panel stays visible at a collapsed size (sidebar-nav pattern)
- Controlled and uncontrolled modes (`isExpanded` / `defaultExpanded` / `onExpandedChange`)
- `placement` selects the collapse axis (`left`/`right` → inline, `top`/`bottom` → block)
- Default (non-peek) collapse uses the browser's `hidden="until-found"` for find-in-page
- `prefers-reduced-motion` support

## Accessibility

### Keyboard

| Key | Action |
| --- | --- |
| `Enter` / `Space` | Toggle the drawer via the trigger button |
| `Tab` | Move focus into/out of the panel |

### ARIA

- `aria-expanded` on the trigger reflects panel state (via RAC Disclosure)
- `aria-controls` links the trigger to the panel; the panel is `role="group"` and `aria-labelledby` the trigger
- Default (non-peek) collapsed panels use `hidden="until-found"` for find-in-page and are hidden from assistive tech
- Peek panels (`minSize`) stay visible and are **not** `aria-hidden`, since their content remains on screen

## Best Practices

- Use `minSize` + `maxSize` for sidebar patterns where the collapsed rail stays visible.
- Omit `minSize` for classic show/hide sections — RAC handles hiding and find-in-page for you.
- `placement` does not flip in RTL; the drawer's side is decided by document/flex order.
- Don't nest InlineDrawers.
