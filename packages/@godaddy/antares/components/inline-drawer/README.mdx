## Features

- In-flow collapsible built on RAC Disclosure — bring your own trigger via `slot="trigger"`
- Animated by default; `animate={false}` disables the transition
- `placement` selects the collapse axis (`left`/`right` → width, `top`/`bottom` → height)
- Controlled and uncontrolled modes (`isExpanded` / `defaultExpanded` / `onExpandedChange`)
- Accordion collapse uses the browser's `hidden="until-found"` for find-in-page
- `prefers-reduced-motion` support

## Sub-components

`InlineDrawerPanel` is the collapsible content for the accordion pattern — a thin wrapper over RAC `DisclosurePanel` that hides (with `hidden="until-found"`) when collapsed. The trigger is any component you slot in with `slot="trigger"` — there's no dedicated trigger component.

### Sidebar Navigation

A rail whose content stays visible when collapsed is a composition, not a mode: drive `isExpanded` yourself, render always-visible children (no `InlineDrawerPanel`), and let `minSize`/`maxSize` size the rail. ## Accessibility

### Keyboard

| Key | Action |
| --- | --- |
| `Enter` / `Space` | Toggle via the slotted trigger |
| `Tab` | Move focus through the content |

### ARIA

- A `<Button slot="trigger">` gets `aria-expanded` + `aria-controls` wired to the `InlineDrawerPanel` automatically (via RAC Disclosure).
- For the sidebar rail, the toggle is your own control (e.g. a `ToggleButton`) — give icon-only buttons an `aria-label`, and the content stays in the DOM and accessible since nothing is hidden.

## Best Practices

- Accordion → `Button slot="trigger"` + `InlineDrawerPanel`. Sidebar rail → controlled state + always-visible children.
- For the rail, `min-content`/`max-content` size to the content; use pixel `minSize`/`maxSize` if you want a fixed rail.
- `placement` does not flip in RTL; the side is decided by document/flex order.
- Don't nest InlineDrawers.
