## Features

- Horizontal and vertical orientations
- Four completion statuses (`none`, `partial`, `complete`, `error`) with status icons
- Timeline (past / current / future) derived from a single `currentStep` prop
- Auto-generated step numbers (opt out with `hideStepNumbers`)
- Interactive (navigable) and inert (display-only) steps
- Title truncation and RTL support
- Rendered as a `<nav>` landmark with an ordered list for assistive technology

## Customization

- Step numbers (`"N. "`) are prepended automatically — keep them where possible, as user testing shows people recognize the component better with them. Set `hideStepNumbers` to omit them.
- `orientation` switches between `horizontal` (accent bar on top) and `vertical` (accent bar on the leading edge).
- `currentStep` is the single source of truth for the timeline. Omit it for a not-yet-started flow where every step is `future`; pass the step count for a fully completed flow where every step is `past`.
- `status` is independent of timeline and only controls the status icon and (for `error`) the accent bar color.
- Pass `className` to the container or an individual step to extend the styles.

## Accessibility

- The container renders a `<nav>` landmark; set `aria-label` to name it (defaults to `"Progress"`).
- Steps are an ordered list (`<ol>`/`<li>`) so assistive technology conveys position and count.
- The current step is marked with `aria-current="step"`.
- Interactive steps are real buttons: focusable in the natural **Tab** order and activated with **Enter** or **Space**. Status icons are decorative (`aria-hidden`).
- Focus is shown with a visible outline; disabled steps use `0.4` opacity and are not focusable.

## Best Practices

- Drive the timeline from `currentStep` rather than styling individual steps.
- Provide `onPress` only for steps a user can actually navigate to; leave it off for display-only steps.
- Keep titles short — they truncate with an ellipsis when space is tight.
- Always give the container an `aria-label` that describes the flow (e.g. `"Checkout progress"`).

## Troubleshooting

- **No step is highlighted:** `currentStep` is omitted, negative, or out of range. Set it to the active step's 0-based index.
- **A step isn't clickable:** it has no `onPress`, so it's inert by design. Add `onPress` to make it interactive.
- **Titles overflow:** that's expected — titles truncate. Shorten the title.
