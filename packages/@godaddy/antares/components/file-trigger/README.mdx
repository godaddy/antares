## Features

- Turns any pressable component (such as an Antares `Button`) into a file picker trigger.
- Renders a visually hidden `<input type="file">` — no styling of its own.
- Restricts selection by MIME type, allows multiple files, directory selection, and camera capture.
- Resets the input value before every open, so re-selecting the same file still fires `onSelect`.

## Accessibility

`FileTrigger` adds no visuals or ARIA of its own — accessibility comes from the pressable child you provide. Use a component with an accessible name (e.g. an Antares `Button` with clear label text) so assistive technology announces the action. The underlying `<input>` is visually hidden but remains operable by keyboard through the child's press behavior.

## Best Practices

- `acceptedFileTypes` is a best-effort filter for the OS picker — always validate file type and size again in your `onSelect` handler, since it can be bypassed.
- Pair `FileTrigger` with `DropZone` to offer both click-to-browse and drag-and-drop. See the DropZone "File Upload" example.
- Keep the child a single pressable element; `FileTrigger` forwards press behavior to whatever you nest inside it.
