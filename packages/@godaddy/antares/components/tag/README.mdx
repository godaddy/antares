## Overview

Tags are short, colored labels that help users quickly identify the status or category of an item. You'll find them on cards, tables, lists, and navigation menus.

Tags are **read-only** — they display information but cannot be clicked or dismissed.

## Installation

```bash
npm install --save @godaddy/antares
```


### Emphasis

Each emphasis communicates a different meaning through color:

| Emphasis | Meaning |
| -------- | ------- |
| `passive` | General label, no urgency |
| `critical` | Something went wrong or needs immediate attention |
| `warning` | Heads up — may need action soon |
| `success` | All good — active, complete, or verified |
| `info` | Helpful context or additional detail |
| `highlight` | Something new, featured, or on sale |
| `premium` | Paid feature or upgrade available |
| `internal` | Internal use only |
| `neutral` | Subtle, low-emphasis label |

## Design Modes

| Mode | What it looks like | Best for |
| ---- | ------------------ | -------- |
| `filled` | Colored background with contrasting text | Most use cases |
| `inline` | Text only, no background or border | Inside paragraphs or sentences |
| `indicator` | White background with a colored dot | Tables, lists, data-heavy views |

## Accessibility

- Screen readers will announce the tag's text naturally as part of the page content.
- Icons and dots are decorative — they are not read aloud.
- Always include a text label. Color alone is not enough to communicate meaning.

## When to Use

- Showing status on a card or list item (e.g. "Active", "Expired")
- Categorizing content (e.g. "New", "Sale", "Beta")
- Displaying metadata labels (e.g. "Pro", "Free trial")

## When Not to Use

- If users need to click or dismiss it — use a **Chip** or **Button** instead.
- If you need a number count — use a **Badge**.
- If the label is part of a sentence with no visual distinction — just use text.
