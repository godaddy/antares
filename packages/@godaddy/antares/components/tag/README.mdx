## Overview

Tags are short, colored labels that help users quickly identify the status or category of an item. You'll find them on cards, tables, lists, and navigation menus.

Tags are **read-only** — they display information but cannot be clicked or dismissed.

## Installation

```bash
npm install --save @godaddy/antares
```


### Emphasis

Each emphasis communicates a different meaning through color:

| Emphasis | Meaning |
| -------- | ------- |
| `passive` | General label, no urgency |
| `critical` | Something went wrong or needs immediate attention |
| `warning` | Heads up — may need action soon |
| `success` | All good — active, complete, or verified |
| `info` | Helpful context or additional detail |
| `highlight` | Something new, featured, or on sale |
| `premium` | Paid feature or upgrade available |
| `internal` | Internal use only |
| `neutral` | Subtle, low-emphasis label |

## Design Modes

The `design` prop controls the visual treatment:

| `design` | What it looks like | Best for |
| -------- | ------------------ | -------- |
| `filled` | Colored background with contrasting text | Most use cases |
| `inline` | Text only, no background or border | Inside paragraphs or sentences |

> **Indicator** is a separate boolean prop (`indicator={true}`), not a `design` value. It renders a colored dot on a plain background and forces high-contrast colors — useful in tables and data-heavy views. See the [Indicator](#indicator) example above.

## Accessibility

### Screen readers

Tags are `<span>` elements with no special ARIA role. Screen readers announce the tag's text as part of normal page content — no extra markup is needed.

### Decorative icons

Icons passed as children must include `aria-hidden="true"` so they are skipped by screen readers. The text label is the sole carrier of meaning.

```tsx
<Tag emphasis="critical">
  <Icon icon="alert" aria-hidden="true" />
  Expired
</Tag>
```

### Indicator dot

The colored dot is a CSS `::before` pseudo-element with no text content. It is invisible to screen readers by design. The text label must convey the status on its own.

## Best Practices

- Use `indicator` for dense layouts (tables, dashboards) where a full colored background would be distracting. Reserve `filled` for prominent placements like cards or hero sections.
- Always mark decorative icons with `aria-hidden="true"` — the text label carries the meaning, not the icon.
- Default to `size="sm"` in tables and compact lists. Step up to `md` or `lg` only when the tag is a primary visual element.
- Use `design="inline"` only within running text. Avoid it in standalone or grouped tag contexts where the background provides useful visual separation.
- Do not combine multiple tags of different emphasis on the same item. Choose the most important one — stacking emphasis dilutes all of them.

## When to Use

- Showing status on a card or list item (e.g. "Active", "Expired")
- Categorizing content (e.g. "New", "Sale", "Beta")
- Displaying metadata labels (e.g. "Pro", "Free trial")

## When Not to Use

- If users need to click or dismiss it — use a **Chip** or **Button** instead.
- If you need a number count — use a **Badge**.
- If the label is part of a sentence with no visual distinction — just use text.
