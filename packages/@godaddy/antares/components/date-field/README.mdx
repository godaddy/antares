## Features

- **Calendar-first**: clicking the field opens a `Calendar` popover; the selected date is shown as a localized value (e.g. `12 Aug 2022`)
- **Optional keyboard entry**: set `allowsKeyboardInput` for editable segments with an icon-only calendar trigger
- **Localized display**: the value is formatted with `useDateFormatter`, ordered per locale; override with `formatOptions`
- **Bounds and availability**: `minValue` / `maxValue` and `isDateUnavailable` pass straight through to the input and calendar
- **React Aria integration**: built on React Aria `DatePicker` for accessibility, forms, and locale handling

## Working with dates

`DateField` (and `DateRangeField`, `Calendar`, `RangeCalendar`) are typed for **`CalendarDate`**
from [`@internationalized/date`](https://react-spectrum.adobe.com/internationalized/date/), a
calendar-and-locale-aware date type — **not** the JavaScript `Date` object. This avoids timezone
and parsing pitfalls.

`@internationalized/date` ships transitively with `@godaddy/antares`, so there's usually nothing
to install. (Under strict pnpm you may need to add it explicitly to your app's dependencies.)

```tsx
import { type CalendarDate, parseDate, today, getLocalTimeZone } from '@internationalized/date';
import { DateField } from '@godaddy/antares';

// A fixed date
<DateField label="Start date" defaultValue={parseDate('2024-03-15')} />;

// Today
<DateField label="Start date" defaultValue={today(getLocalTimeZone())} />;

// Controlled
function Controlled() {
  const [value, setValue] = useState<CalendarDate | null>(null);
  return <DateField label="Start date" value={value} onChange={setValue} />;
}
```

Wrap your app in `<I18nProvider locale="…">` to control locale and date ordering. See the
[React Aria DatePicker docs](https://react-spectrum.adobe.com/react-aria/DatePicker.html) and the
[`@internationalized/date` docs](https://react-spectrum.adobe.com/internationalized/date/) for more.
