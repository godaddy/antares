## Features

- **Multiple series**: Shows all series that have a datum at the hovered X position in a single tooltip.
- **Series matching**: Rows are built from `series` by matching `series[].id` to keys in `tooltipData.datumByKey` (from visx at the hovered X).
- **Value formatting**: You can supply a function to format each value; if you don't, the tooltip shows the point's `y` (or nothing when null). Use the formatter for custom data shapes.

## Internal Component

This component is **internal-only** and not exported from the package. It is intended for use inside chart components.

## Value formatting

**Default:** When `formatValue` is not provided, the tooltip uses a formatter that reads the data point’s `y` property (same as the chart's default Y accessor), converts it to a string, and returns an empty string for null. This is correct for the default `DataPoint` shape (`{ x, y }`).

**Custom:** Pass `formatValue` to control display. The function receives the datum and returns a string. Use it to both extract and format (e.g. currency, percentage, custom units). For custom data point shapes (generic `TooltipProps<YourDatum>`), you must pass `formatValue` because the default only reads `y`.

```tsx
<Tooltip
  tooltipData={tooltipData}
  series={series}
  formatValue={(d) => `$${d.y?.toLocaleString() ?? ''}`}
/>
```
