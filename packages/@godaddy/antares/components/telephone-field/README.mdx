## Features

- **Country picker + phone input**: A country/dial-code `Select` and a phone number input share one bordered field
- **Label, description, error**: Optional label, helper text, and error message with proper accessibility
- **Controlled or uncontrolled**: Use `country`/`onCountryChange` and `value`/`onChange` for controlled state, or `defaultCountry`/`defaultValue` for uncontrolled
- **Validation states**: Use `isInvalid` with `errorMessage` and `isDisabled` for validation and disabled state
- **Raw values, no formatting**: The phone number is the exact string the user typed — TelephoneField applies no masking or E.164 formatting, so combine `country` and `value` however the consuming form or backend expects
- **Composable options**: Country options are plain `SelectItem` children, so callers control the exact list and label of supported countries

## Accessibility

- The country picker gets its own accessible name from `countryLabel`, since it has no visible label of its own — always pass one, localized for your UI
- For anything else the country picker needs (`aria-describedby`, `aria-labelledby`, `id`, …), pass `countryProps`, which is spread onto its underlying `FieldSelect`
- `label`, `description`, and `errorMessage` are associated with the field automatically
- Built on React Aria's `TextField` and `Select`, giving full keyboard support: `Tab` moves between the country picker and the phone input, `Enter`/`Space` opens the country picker, and arrow keys/typeahead navigate its options
- `isDisabled` and `isInvalid` are reflected with `aria-disabled`/`data-disabled` and `aria-invalid`/`data-invalid` on both segments

## Best Practices

- Keep country option labels short and consistent (e.g. always `<ISO code> +<dial code>`) so the trigger stays readable at both `md` and `sm` sizes
- Pair `isInvalid` with a specific `errorMessage` (e.g. "Enter a valid phone number") rather than a generic one
- Validate and format the combined `country` + `value` where the number is consumed (on submit, or server-side) — TelephoneField intentionally does not mask or format as the user types
