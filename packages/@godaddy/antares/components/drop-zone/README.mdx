### File Upload

A reference showing how to compose `DropZone` with other Antares components to build a functional file upload experience. Use this as a starting point — adapt the accepted formats, size limit, and handlers to your product's requirements.

It demonstrates the two common upload affordances working together:

- **Click to browse** — the "Add files" button opens the native file picker and returns the selection through `onSelect`. `acceptedFileTypes` restricts the picker to the supported formats.
- **Drag and drop** — `getDropOperation` inspects the dragged types and returns `'cancel'` for unsupported files so the zone won't accept them, while `onDrop` reads the dropped files with the `isFileDropItem` guard.

Both paths funnel into one handler that renders a preview. `acceptedFileTypes` and `getDropOperation` are best-effort UX filters — validate file type and size again in your handler, since they can be bypassed.

When the drop zone content is composed from multiple elements (icon, button, text), pass `aria-label` directly on `DropZone` to provide a single accessible name for the hidden drop button.

### Drop Target Label

Pass a render-prop function as `children` to change the zone's content while a valid drag is active. RAC invokes it with [`DropZoneRenderProps`](https://react-spectrum.adobe.com/react-aria/DropZone.html#dropzonerenderprops) on every state transition, so branching on `isDropTarget` requires no extra state. Pair it with `getDropOperation` to reject unsupported types — the label only appears when the drag is actually accepted.

### Replace Existing File

Demonstrates the `isFilled` pattern: when the zone already holds a file, dragging a new valid file over it shows a "Drop to replace" label while the container switches to its active teal state. The drag path and the click path (`FileTrigger`) share the same handler, so both replace the current selection.

`isFilled` is not a built-in prop — derive it from your own state so your product controls when the zone is considered occupied and when it resets.

## Accessibility

### Keyboard

| Key | Description |
| --- | --- |
| `Tab` | Moves focus to the drop zone |
| `Enter` / `Space` | Opens the file picker so files can be selected without a pointer |

### Screen readers

The drop zone renders a visually hidden `<button>` inside the container. Assistive technology discovers and announces this button as an interactive element — activating it opens the native file picker without a pointer.

The button's accessible name is built from two sources, combined in order:

1. **`<Text slot="label">`** — any `Text` child with `slot="label"` has its `id` linked to the button via `aria-labelledby`. This is the recommended way to provide a contextual label that is also visible to sighted users.
2. **`aria-label`** — pass `aria-label` on the `DropZone` component to override or supplement the label (e.g. `aria-label="Upload profile photo"`). When no `Text slot="label"` is present and no `aria-label` is set, the button falls back to the default "DropZone" label provided by React Aria.

```tsx
<DropZone onDrop={handleDrop}>
  <Text slot="label">Drop files to upload.</Text>
</DropZone>
```

## Best Practices

- Use the drop zone alongside a button-based file picker — drag-and-drop is a convenience, never the only way to upload.
- Replace the default text with `children` to communicate the accepted file types and size limits up front.
- Let the drop zone fill its container; constrain it with the surrounding layout rather than setting a fixed width on the component itself.
- Reflect upload state (validating, uploading, error) outside the drop zone so the target area stays clear and predictable.
