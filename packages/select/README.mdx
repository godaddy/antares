# Select

The `@bento/select` package provides an accessible, customizable dropdown that overcomes native `<select>` limitations. It exports the **Select** primitive, enabling you to build flexible select dropdowns with slot-based composition where Select acts as coordinator, applying necessary props to slotted children.

Users can slot custom components (Button, Popover, ListBox, etc.) for maximum flexibility while maintaining accessibility through React Aria integration.

## Installation

```bash
npm install --save @bento/select
```

## Slot Map

The `Select` primitive uses a slot-based architecture where you provide child components with specific slot names. The Select component coordinates props to these slotted children.

| Slot Name   | Component Type | Description                                | Required |
| ----------- | -------------- | ------------------------------------------ | -------- |
| `trigger`   | Button-like    | The button that opens the select dropdown  | Yes      |
| `label`     | Label-like     | Accessible label for the select            | No       |
| `value`     | Text-like      | Displays the currently selected value      | No       |
| `trigger.value` | Text-like  | Alias for `value` slot (enables nesting inside trigger) | No |
| `popover`   | Popover-like   | The overlay container for the options list | Yes      |
| `listbox`   | ListBox        | The list of selectable options             | Yes      |
| `description` | Text         | Optional description text                  | No       |
| `error`     | Text           | Optional error message                     | No       |

## Data Attributes

The `Select` primitive provides the following data attributes for styling:

| Attribute           | Description                                    |
| ------------------- | ---------------------------------------------- |
| `data-open`         | Indicates the select dropdown is open          |
| `data-disabled`     | Indicates the select is disabled               |
| `data-focused`      | Indicates the select is focused                |
| `data-focus-visible`| Indicates focus ring should be shown           |
| `data-hovered`      | Indicates the select is hovered                |
| `data-invalid`      | Indicates the select has validation errors     |
| `data-required`     | Indicates the select is required               |
| `data-empty`        | Indicates the collection is empty              |

## Selection Modes

The Select component supports two selection modes:

- **single** (default): Allows selecting one value. The value prop/state is `Key | null`.
- **multiple**: Allows selecting multiple values. The value prop/state is `Key[]` (array of keys).
